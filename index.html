Workflow / Reading method / UI logic

01

首先导入文档内容。题目文档需要手动调参，文档内容可以直接在机考网复制过来；遇到无法复制的题型如标题题，Mac 可以采用 command + shift + 4 截图提取文字，或使用微信截图识取文字提取。

02

选中文本按 A，标记主干句。

03

选中文本按 S，标记修饰句。

04

选中文本按 W 搜索单词并加入生词本 / 错词本；查词模块按百度翻译搜索模型的产品逻辑设计。

05

标记的文本按 D，选择并标记词块论证标签。

06

长按 Shift，依次选择标记块 A → B，进行逻辑连线。

07

双击高亮文本，取消高亮。

08

主要目的：基于哔站迷糊老师的长难句阅读法思路，用于培养英文脑、论证脑的阅读习惯。用词块思维和词块关系去理解文章，摒弃“抓词”的思路，转而抓句子层次、段落层次和词块层次，强调论证过程与写作思路，克服中文脑走马观花式阅读，避免读到最后感觉白读。

说明文档：查词模块用于快速搜索单词并沉淀到生词本；当前前端部署版使用公开翻译接口，若要正式接入百度翻译官方 API，需要后续配置 API Key 与后端签名。

设计说明 · 使用文档

快捷键、使用流程与阅读训练目标。

导入
先导入文档内容题目文档需要手动调参；可直接从机考网站复制。标题题等无法复制时，可用 Mac 的 command + shift + 4 截图提取，或微信截图识取文字。
A
主干句选中文本后按 A，用低饱和红色高权重字体标记论证主干、核心观点、主谓宾结构。
S
修饰句选中文本后按 S，用低饱和绿色底纹标记限定、补充、举例、背景等修饰性信息。
W
搜索单词并加入生词本选中单词后按 W，即可查询并保存到生词本 / 错词本。查词模块按百度翻译搜索模型的产品逻辑设计；当前前端部署版使用公开翻译接口，如需正式接入百度翻译官方 API，需要后续配置 API Key 与后端签名。
D
词块论证标签给已标记文本添加观点、证据、对比、原因、结果、同义替换等词块标签。
Shift
逻辑连线长按 Shift，依次点击两个已标记块 A → B，建立逻辑连线。
双击
取消高亮双击已标记的文本，即可移除高亮和其关联逻辑线。
训练目标：基于哔站迷糊老师的长难句阅读法思路，培养英文脑、论证脑阅读习惯。核心是摒弃“抓词”的阅读惯性，转而抓句子层次、段落层次和词块层次，理解论证过程和写作思路，克服中文脑走马观花式阅读，避免读到最后感觉白读。
